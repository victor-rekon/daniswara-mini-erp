# Client Data Checklist

Request these before serious implementation and prototype validation.

## Operational Data

1. Sample production data
2. Sample losses / susut data
3. HPP formula currently used
4. Product list
5. Customer list
6. Branch list
7. User list and roles
8. Sample quotation
9. Sample customer PO / SO
10. Sample surat jalan
11. Sample invoice
12. Sample payment report
13. Sample weekly owner report

## Accounting Light Data

1. Chart of accounts / daftar akun
2. Expense categories
3. Journal entry examples
4. P&L format expected by management
5. Confirmation that P&L is management report only, not official tax/audit report

## Confirmation Needed

- HPP formula owner
- Losses categories
- Payment status options
- Delivery status options
- Invoice overdue rule
- Whether staff may edit/delete records
- Whether owner is read-only
- Whether Malaysia branch is only branch filter or needs separate access visibility

## Data Sensitivity Rule

Client may censor sensitive values. SistemBeres needs the structure, column names, workflow, and sample logic more than exact private numbers.
