# Prototype Plan

## Prototype Strategy

Deliver the operational frame first, then accounting light. Do not attempt full ERP behavior.

## Target 7-Day Operational Prototype

### Day 1

- Project setup
- App shell
- Sidebar
- Supabase schema draft
- Login placeholder / auth plan

### Day 2

- Master data frame
- Products
- Customers
- Branches
- Users

### Day 3

- Production input
- Losses input
- Basic HPP fields
- Production table

### Day 4

- Quotation
- Customer PO / SO tracking
- Sales input

### Day 5

- Surat jalan / delivery
- Invoice
- Customer payment
- Outstanding calculation

### Day 6

- Owner dashboard
- Latest records
- Pending delivery
- Outstanding customer
- Overdue invoice

### Day 7

- Cleanup
- Dummy data test
- Mobile layout check
- Prototype walkthrough notes

## Accounting Light Extension: Additional 3–7 Working Days

- Chart of accounts
- Manual journal entries
- Expense input
- Debit/credit check
- Simple P&L management report

## Acceptance Criteria

- User can navigate all Phase 1 pages
- User can input master data
- User can input production/losses/HPP data
- User can create quotation
- User can record customer PO/SO and surat jalan
- User can create invoice and update customer payment
- Owner can view operational summary
- Accounting light can show manual journal and simple P&L once COA is confirmed

## Non-Acceptance Items

Do not judge prototype by:

- Full accounting completeness
- Tax readiness
- Bank reconciliation
- Vendor payable
- Inventory valuation
- Mobile native app behavior
- Complex approval workflow
